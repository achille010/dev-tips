# Contributing to dev-tips

First off — thanks for wanting to help! This repo grows by people like you sharing what they know.

---

## How to contribute

1. **Fork** this repo and create a new branch (`git checkout -b my-new-tip`)
2. **Add your content** in the right folder (see the guide below)
3. **Update the index** — add a link to your file in the folder's `README.md`
4. **Open a Pull Request** with a short description of what you added

---

## Where does my content go?

| Type of content | Folder |
|---|---|
| A quick tip (under ~300 words) | `/tips` |
| A walkthrough or tutorial (longer) | `/guides` |
| A concept explanation (language-agnostic) | `/concepts` |
| A debugging guide or error explanation | `/debugging` |
| A tool, extension, or productivity tip | `/tools` |
| A useful link, book, or cheatsheet | `/resources` |

---

## File naming

Use lowercase with hyphens. Be descriptive but concise.

✅ `git-rebase-vs-merge.md`  
✅ `how-to-read-a-stack-trace.md`  
❌ `MyTip.md`  
❌ `tip1.md`

---

## Content guidelines

- **Be clear and direct.** Write like you're explaining to a teammate.
- **Include examples.** Code snippets, commands, and before/after comparisons go a long way.
- **Keep it language-agnostic where possible.** If it's language-specific, say so upfront.
- **Link your sources** if you're building on someone else's work.
- **No self-promotion or ads.**

---

## Markdown style

- Use `#` for the title, `##` for sections
- Use fenced code blocks with a language tag (` ```bash `, ` ```python `, etc.)
- Keep lines reasonably short (80–100 chars) for readability in raw view

---

## Not sure where to start?

Check the open [Issues](../../issues) for things people want covered, or just add something you wish you'd known earlier.
